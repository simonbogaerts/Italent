.NET - “RESTful API”

2018 @ Pluralsight


Iedere programmeur heeft zo wel zijn favoriete programmeertaal, in mijn geval is dat zonder enige twijfel .NET (dotnet). Dit is de taal waarin je voelt dat je alle opdrachten aan kan, en waarin je ook het meeste plezier hebt om te werken. In vele gevallen groeit een taal ook doorheen de jaren. Java zit zo bijvoorbeeld al aan versie 9, of Python aan versie 3.

Maar wat heel weinig gebeurd is dat een taal of framework helemaal opnieuw vanaf de grond herbouwd wordt met een totaal andere mindset. Maar in het geval van dotnet is dit wel het geval! Enkele jaren geleden is het team verantwoordelijk voor dotnet veranderd, en met een nieuw team kwam ook een nieuwe mindset. In tegenstelling tot de oude dotnet, is dotnet core helemaal anders, open source, multi-platform en bedoeld om mee te kunnen draaien met de beste frameworks op de markt.

Tijdens mijn opleiding heb ik enkel de oude versie, asp.NET gezien. Dit terwijl dot net core, en zelfs dot net core 2.0 eigenlijk al uit waren op dit moment. Dot net core 2.0 kan vrijwel alles dat de asp.NET kon, alleen op een veel elegantere, doordachte en betere manier. Om dotnet core uit te testen heb ik daarom besloten om mij te verdiepen in de wereld van REST, vele malen dieper dan alles wat ooit in mijn opleiding aan bod is gekomen.

Ik heb hierbij gebruik gemaakt van een Pluralsight cursus gemaakt door Microsoft MVP, Kevin Dockx. In deze cursus wordt stap voor stap uitgelegd hoe je een echte level 3 RESTful server ontwikkeld die volledig voldoet aan de vereiste gesteld door Roy Fielding, de originele ontwikkelaar van REST.


Verslag

Het eerste hoofdstuk van deze cursus legt dan ook in uitzonderlijk veel details uit wat een RESTful server dan ook precies is en aan welke voorwaarde voldaan moet zijn om hierover te kunnen spreken. Het Richardson Maturity Model, waarmee de maturiteit van de REST server beoordeeld wordt, wordt hier ook besproken.

Vervolgens wordt de basis functionaliteit voorzien voor de standaard http methodes zoals GET, POST, PUT, PATCH, en DELETE. Hierbij wordt er heel veel aandacht besteed aan het belang van status codes en Data Transfer Objects (DTO).

En dan wordt er overgegaan naar het echte werk. Validatie is het eerste grootte onderdeel dat besproken wordt, en dit in detail voor POST, PUT en PATCH apart. In hetzelfde hoofdstuk wordt ook logging uit de doeken gedaan, en hoe je dit makkelijk kan integreren met de rest van dotnet core.

De volgende twee hoofdstukken zijn een uitbreiding op de GET functionaliteit. Paging, filtering en searching worden geïmplementeerd en meer geavanceerde functionaliteit zoals query based filtering en data shaping komen aan bod. Dit allemaal samen zorgt voor een zeer professionele en krachtige REST server.

De volgende hoofdstukken nemen de andere REST beperkingen op zich, namelijk “Hypermedia As Engine Of Application State” (HATEOAS), cashing en concurrency.

De toepassing die ik ontwikkeld heb door de kennis van deze cursus om te zetten in praktijk kan gevonden worden op de volgende url.


Reflectie

Eén van de belangrijkere momenten en een sleutelpunt uit mijn educatieve loopbaan. Deze cursus heeft er eigenhandig voor gezorgd dat ik nu dotnet mijn favoriete programmeertaal noem. Ik heb gedurende mijn periode op de PXL vele malen gebruik gemaakt van Pluralsight maar deze cursus is veruit de beste die ik behandeld heb gedurende deze periode. Uiterst leerrijk, ongelofelijk gedetailleerd en duidelijk gebracht en een absolute meerwaarde. Ik heb niet alleen mijn kennis omtrent REST vergroot, maar mijn algemene kennis over dotnet is er ook op ongelofelijk op vooruit gegaan.

Vele concepten die behandeld zijn geworden tijdens mijn opleiding zijn nu pas echt duidelijk geworden. Ik denk dan al snel aan het gebruik van IQueryables of het gebruik van extension methods, twee concepten waar ik wel al van gehoord had, maar moeite mee had om een effectieve real-life use-case voor te verzinnen. Beide van deze concepten zijn meermaals aan bod gekomen en hebben duidelijk gemaakt waarom ze nu juist zo handig zijn.

Entity Framework en het repository pattern hebben na het volgen van deze cursus ook heel wat minder geheimen voor mij. Vroeger wist ik hoe deze te gebruiken, terwijl ik nu kan zeggen dat ik weet waar ik echt mee bezig ben.

Ik heb echter niet alleen kennis bijgewerkt tijdens het volgen van deze cursus, neen ik heb hoofdzakelijk nieuw kennis op gedaan. Deferred execution, marker interfaces, object mappers, data transfer objecten en in het bijzonder expando objects waren volledig nieuw voor mij. Ik ben ervan overtuigd elk van deze nog vele malen tegen te komen in mijn verdere loopbaan.

Tijdens het volgen van deze cursus ben ik op problemen gestoten, reële problemen waarvan ik de oplossing ook niet snel meer zal vergeten. Voorbeelden hiervan zijn het gebruik van een multi-layered projectstructuur in dotnet core of het automatische seeden en migreren van een Entity Framework core database. problemen die niet van toepassing waren in de cursus en waardoor ik zelf op zoek ben moeten gaan naar oplossingen.

Ik heb geleerd dat ik veel gebruik maakten vroeger van technologieën zonder te weten hoe deze werkte, of waarom deze nu gebruikt werden. Deze realisatie en het algemene leerproces hebben mij overtuigd dit onderdeel op te nemen in mijn portfolio.
